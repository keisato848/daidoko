## 3. バックアップ・復元

### 方針

SQLite の全主要テーブルを JSON にシリアライズし、アプリの document 領域に保存する。復元は同じ形式の最新バックアップを読み込み、ローカル DB を置き換える。

### 保存先

- `FileSystem.documentDirectory/backups/`
- ファイル名: `daidoko-backup-YYYYMMDD-HHmmss.json`

### バックアップ対象

以下の通常テーブルを対象とする。FTS 仮想テーブルは復元後に再構築する。

- `users`
- `families`
- `family_members`
- `sources`
- `recipes`
- `recipe_revisions`
- `ingredients`
- `steps`
- `tags`
- `recipe_tags`
- `cooking_logs`
- `cooking_photos`
- `memos`
- `sync_meta`
- `app_meta`

### JSON 形式

```json
{
  "format": "daidoko.local-backup",
  "schemaVersion": 1,
  "exportedAt": "2026-05-30T00:00:00.000Z",
  "tables": {
    "users": [],
    "families": []
  }
}
```

### 復元手順

1. 最新バックアップを選択する。
2. `format` と `schemaVersion` を検証する。
3. `PRAGMA foreign_keys = OFF` の状態で対象テーブルを依存順と逆順に削除する。
4. 依存順に insert する。
5. `PRAGMA foreign_keys = ON` に戻す。
6. `rebuildFts()` で検索インデックスを再構築する。
7. アプリ内状態は画面再読み込みで反映する。

### UI

設定画面の「バックアップ・復元」を有効化し、専用画面で以下を提供する。

- バックアップ作成
- 最新バックアップから復元
- 最新バックアップの作成日時とサイズ表示
- 復元前の確認ダイアログ

## 5. 調理記録写真追加

### 方針

既存の `cooking_photos` テーブルと `createCookingLog(input.photos)` を利用する。カメラ/ギャラリーから選択した画像は一時 URI のまま保存せず、アプリ document 領域へコピーしてから DB に保存する。

### 保存先

- `FileSystem.documentDirectory/cooking-photos/`
- ファイル名: `cooking-photo-{timestamp}-{random}.{ext}`

### UI

調理記録画面で以下を提供する。

- カメラで撮影
- ギャラリーから選ぶ
- 選択済み写真のサムネイル表示
- 写真削除
- 最大 6 枚まで
- 権限拒否/キャンセル時は画面を維持してメッセージ表示

### 保存

`createCookingLog()` に `photos` を渡す。

```ts
{
  localPath: copiedImagePath,
  takenAt: capturedAt
}
```
