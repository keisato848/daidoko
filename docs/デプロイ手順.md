# だいどこ — デプロイ手順

> 対象: v1.0 Option A（Railway サーバー + EAS TestFlight）  
> 前提: Node.js 20 / pnpm 9 / Xcode インストール済み

---

## Part 1 — サーバーデプロイ（Railway）

### 1-1. Railway CLI ログイン

```powershell
cd C:\Projects\daidoko
railway login
```

ブラウザが開くので GitHub アカウントでサインイン。  
ターミナルに `Logged in as ...` が出たら完了。

```powershell
# 確認
railway whoami
```

---

### 1-2. Railway プロジェクト作成 & デプロイ

```powershell
# プロジェクト初期化（対話式: "Create a new project" を選択）
railway init

# デプロイ実行（ルートの Dockerfile を使用）
railway up --detach
```

> `--detach` はビルドをバックグラウンドで走らせてターミナルを返します。  
> ビルドログは Railway ダッシュボードで確認できます。

```powershell
# デプロイ状況確認
railway status

# ブラウザでダッシュボードを開く
railway open
```

---

### 1-3. 公開 URL の確認

Railway ダッシュボードの **Deployments → Domain** に表示される URL をコピーします。

```
例: https://daidoko-server-production.up.railway.app
```

ヘルスチェックで動作確認：

```powershell
# URL を実際のものに置き換えて実行
$SERVER_URL = "https://daidoko-server-production.up.railway.app"
Invoke-WebRequest -Uri "$SERVER_URL/health" | Select-Object -ExpandProperty Content
# → {"status":"ok","ts":"..."}  が返れば成功
```

---

### 1-4. Railway の環境変数設定

```powershell
# PORT はデフォルト 3000。Railway が上書きするので設定不要
# 必要に応じて追加変数を設定する場合
railway variables set NODE_ENV=production
```


実行後、`app.json` の `extra.eas.projectId` に UUID が書き込まれます。

```powershell

git commit -m "chore: set EAS projectId"
git push origin develop
```

---

### 2-3. 環境変数（サーバー URL）の設定

Part 1-3 で取得した Railway URL を設定します。

```powershell
cd C:\Projects\daidoko\apps\mobile

# .env.local を作成（Git 管理外）
Copy-Item .env.example .env.local

# .env.local を編集して Railway URL に書き換え
# EXPO_PUBLIC_SERVER_URL=https://daidoko-server-production.up.railway.app
notepad .env.local
```

EAS Build のクラウドビルドにも環境変数を渡します：

```powershell
eas env:create --scope project --name EXPO_PUBLIC_SERVER_URL --value "https://daidoko-server-production.up.railway.app" --environment preview
eas env:create --scope project --name EXPO_PUBLIC_SERVER_URL --value "https://daidoko-server-production.up.railway.app" --environment production
```

## Part 3 — 動作確認チェックリスト

サーバー・アプリ両方が動いたら以下を確認してください。

### サーバー

- [ ] `GET /health` → `{"status":"ok"}`
- [ ] `POST /api/v1/import/url` にクラシルの URL を送って RecipeDraft が返る

```powershell
$SERVER_URL = "https://daidoko-server-production.up.railway.app"
Invoke-RestMethod -Method Post -Uri "$SERVER_URL/api/v1/import/url" `
  -ContentType "application/json" `
  -Body '{"url":"https://www.kurashiru.com/recipes/f52c6f7c-7f28-4d93-b8c6-e1a4e99d2ae6"}'
```

### モバイルアプリ（TestFlight）

- [ ] ホーム画面・タイムラインが表示される
- [ ] レシピ一覧に 6 件のシードデータが表示される
- [ ] 「＋ 追加」→「URLから取り込み」でレシピを取り込める
- [ ] 手動でレシピを作成・保存できる
- [ ] 料理中モードで手順を進められる
- [ ] 料理完了後に評価・メモを記録できる

## トラブルシューティング

| 症状                            | 原因と対処                                                                                 |
| ------------------------------- | ------------------------------------------------------------------------------------------ |
| `railway up` でビルドエラー     | Railway ダッシュボードのビルドログを確認。`Dockerfile` のパスが `/` 起点になっているか確認 |
| ヘルスチェックが返らない        | Railway のドメイン設定が完了するまで 1〜2 分かかる                                         |
| EAS ビルドで `projectId` エラー | `eas build:configure` を再実行して `app.json` に ID を設定                                 |
| TestFlight に届かない           | App Store Connect → TestFlight → ビルド処理中（最大 30 分）待つ                            |
| URL取り込みが `FETCH_FAILED`    | Railway のログを確認。外部サイトへの fetch が Railway のプランで制限されていないか確認     |
| レシピが保存されない            | デバイスの SQLite は正常動作のはず。アプリを再起動してシードデータが表示されるか確認       |
