## 2. ユーザーストーリー詳細

### 2.1 レシピ登録

---

**US-01 URL 取り込み**

> As P1（主調理担当者）,  
> I want to paste a recipe URL and have the title, ingredients, and steps auto-extracted,  
> So that I can save a recipe from external sites without retyping.

**受け入れ基準（ハッピーパス）**

- [ ] URL 入力欄に貼り付けて「読み込む」を押すと、タイトル・材料・手順がプレビュー表示される
- [ ] JSON-LD Recipe schema が存在するサイト（クラシル、クックパッド等）で成功率 ≥ 85%
- [ ] プレビューを確認して「蔵書に追加」で保存できる
- [ ] 保存後、蔵書一覧にレシピが表示される

**エッジケース・失敗系**

- [ ] URL が無効（404 / タイムアウト）: エラーメッセージを表示し、手動入力への誘導を表示する
- [ ] JSON-LD が存在しないサイト: 「自動取り込みできませんでした。手動で入力しますか？」とフォールバック案内を表示する
- [ ] 取り込み中に通信が切れた場合: スピナーを止め、再試行ボタンを表示する
- [ ] タイトルは取れたが材料・手順が空の場合: 部分的な取り込み結果を表示し、ユーザーが補完できる

---

**US-02 写真取り込み（OCR）**

> As P1（主調理担当者）,  
> I want to photograph a handwritten recipe or cookbook page,  
> So that I can digitize analog recipes without manual typing.

**受け入れ基準（ハッピーパス）**

- [ ] カメラ起動 → 撮影 → OCR 処理 → テキスト確認画面の順で遷移する
- [ ] 認識されたテキストがタイトル・材料・手順に自動振り分けされる（信頼度が低い箇所はハイライト）
- [ ] ユーザーが確認・修正してから保存できる

**エッジケース・失敗系**

- [ ] 画像が暗い・ブレている: 「読み取りにくい画像です。再撮影しますか？」と案内
- [ ] 手書き文字の認識精度が低い: 認識できなかった箇所を空欄にして手動入力を促す
- [ ] OCR エンジンがオフライン非対応の場合: 「この機能はインターネット接続が必要です」と表示（v1 実装方針次第）

---

**US-03 手動入力**

> As P1（主調理担当者）,  
> I want to manually enter a recipe's title, ingredients, and steps,  
> So that I can record original or family-passed-down recipes in my own words.

**受け入れ基準（ハッピーパス）**

- [ ] タイトル・副題・カテゴリ・人数・時間・写真・材料（複数行）・手順（複数行）・覚書・タグを入力できる
- [ ] 材料・手順は「＋追加」で行を動的に増やせる
- [ ] 写真はカメラ撮影またはライブラリから選択できる
- [ ] 途中保存（下書き）ができる
- [ ] 保存すると蔵書一覧に反映される

**エッジケース・失敗系**

- [ ] タイトルが空のまま保存しようとした場合: バリデーションエラーを表示し、フォームを閉じない
- [ ] 写真のファイルサイズが上限を超えた場合: 自動リサイズするか「この写真は使用できません」と案内
- [ ] 入力途中でアプリを閉じた場合: 下書きとして自動保存し、次回起動時に「下書きを続ける？」を表示

---

**US-04 レシピのバージョン派生**

> As P1 or P2（家族メンバー）,  
> I want to fork an existing recipe and save it as a variant,  
> So that I can record my own arrangement without overwriting the original.

**受け入れ基準（ハッピーパス）**

- [ ] レシピ詳細画面から「派生バージョンを作成」が選択できる
- [ ] 元レシピのデータがコピーされた状態で編集フォームが開く
- [ ] バージョン名（例: 「夏野菜入り」「味噌少なめ」）をつけて保存できる
- [ ] 元レシピの詳細画面にバージョン一覧が表示される

**エッジケース・失敗系**

- [ ] 家族メンバー（P2）が他人のオリジナルを派生させる場合: 派生は可能、ただし「オリジナルの編集は不可」を明示
- [ ] バージョンをさらに派生（孫レシピ）させる場合: 2 段階まで許可、それ以上は「オリジナルから派生してください」と案内

---

### 2.2 レシピ閲覧・検索

---

**US-05 蔵書一覧の閲覧**

> As P2（家族メンバー）,  
> I want to browse and filter the recipe library,  
> So that I can find the recipe I want to cook today.

**受け入れ基準（ハッピーパス）**

- [ ] カテゴリ・タグ・フリーワードで絞り込みができる
- [ ] 調理回数・最終調理日・登録日でソートできる
- [ ] 「作りたい」フラグ付きレシピのみ表示するフィルターがある
- [ ] 300 件まで 200ms 以内に表示される

**エッジケース・失敗系**

- [ ] 検索結果が 0 件: 「見つかりませんでした。レシピを追加しますか？」と案内
- [ ] 蔵書が空の状態: オンボーディング UI（「最初のレシピを追加しましょう」）を表示

---

**US-06 レシピ詳細の閲覧**

> As P2（家族メンバー）,  
> I want to view all details of a recipe on a single screen,  
> So that I can understand what ingredients and steps I need.

**受け入れ基準（ハッピーパス）**

- [ ] タイトル・副題・カテゴリ・人数・時間・写真・材料・手順・覚書・調理履歴が表示される
- [ ] バージョン一覧が表示され、タップで切り替えられる
- [ ] 「料理中モード」と「買い物モード」へのボタンがある
- [ ] 「作った！」ボタンが目立つ位置にある

---

### 2.3 調理サポート

---

**US-07 料理中モード**

> As P2（家族メンバー）,  
> I want to view each cooking step in full-screen with large text and swipe navigation,  
> So that I can follow the recipe hands-free while cooking.

**受け入れ基準（ハッピーパス）**

- [ ] 手順が 1 ステップずつ全画面大文字で表示される
- [ ] 左右スワイプ（または画面端タップ）で前後に移動できる
- [ ] 現在のステップ番号と総ステップ数が表示される
- [ ] 画面が常時 ON になる（スリープしない）
- [ ] 終了ボタンで詳細画面に戻れる

**エッジケース・失敗系**

- [ ] 手順が 1 つしかない場合: スワイプ不要でその 1 ステップのみ表示
- [ ] モードを抜けた後: スリープ設定を元に戻す
- [ ] テキストが非常に長いステップ: スクロール可能にする（スワイプとの競合を考慮）

---

**US-08 買い物モード**

> As P1 or P2,  
> I want to see only the ingredient list with checkboxes when shopping,  
> So that I can confirm what I need to buy without unnecessary information.

**受け入れ基準（ハッピーパス）**

- [ ] 材料名・分量のリストがチェックボックス付きで表示される
- [ ] チェックを入れると打ち消し線が入り視覚的に管理できる
- [ ] 人数変更で分量が自動スケールされる
- [ ] 複数レシピの材料を合算表示できる

**エッジケース・失敗系**

- [ ] 分量が「適量」「少々」のもの: スケールせずそのまま表示
- [ ] 画面を閉じた場合: セッション中はチェック状態を保持する

---

### 2.4 調理記録

---

**US-09「作った！」記録**

> As P1 or P2,  
> I want to log that I cooked this recipe with one tap,  
> So that I can track when and by whom each recipe was made.

**受け入れ基準（ハッピーパス）**

- [ ] 「作った！」ボタンを押すと現在日時・ログインユーザーが自動設定される
- [ ] 任意で写真・コメントを追加できる
- [ ] 記録後、レシピの「調理回数」と「最終調理日」が即時更新される
- [ ] ホームの「家族の最新調理」セクションにも反映される

**エッジケース・失敗系**

- [ ] 同日に 2 回「作った！」した場合: 別々のログとして保存される
- [ ] 写真添付が失敗した場合: 写真なしで記録を保存し「写真は後で追加できます」と案内
- [ ] オフライン状態でも記録できる（ローカル保存）

---

### 2.5 家族共有

---

**US-10 家族メンバーの招待**

> As P1（オーナー）,  
> I want to invite family members to share my recipe book,  
> So that everyone in the family can access and cook from the same recipes.

**受け入れ基準（ハッピーパス）**

- [ ] 設定画面から「家族を招待」を選択できる
- [ ] QR コードまたは招待コード（6桁）が生成される
- [ ] 招待を受けた側がコードを入力 / QR を読み取ると家族グループに参加できる
- [ ] 参加後、家族全員の蔵書にアクセスできる

**エッジケース・失敗系**

- [ ] 招待コードの有効期限切れ（24 時間）: 「このコードは無効です。新しいコードを発行してください」と案内
- [ ] 同じデバイスで複数アカウントを切り替える場合: 一度に 1 つのグループにのみ所属できる
- [ ] オーナーがアプリを削除した場合: 他メンバーへのデータ引き継ぎフロー（P2 課題として記録）

---

**US-11 権限管理**

> As P1（オーナー）,  
> I want to ensure family members can view and record but not edit or delete my recipes,  
> So that the original recipes are protected from accidental changes.

**受け入れ基準（ハッピーパス）**

- [ ] オーナーのみ: 作成・編集・削除が可能
- [ ] 家族メンバー: 閲覧・「作った！」記録・バージョン派生のみ可能
- [ ] 権限の違いが UI 上で明確（編集ボタン非表示・「閲覧のみ」バッジなど）

**エッジケース・失敗系**

- [ ] 家族メンバーが編集を試みた場合: 「このレシピはオーナーのみ編集できます」とトースト表示
- [ ] オーナーが家族メンバーを削除した場合: そのメンバーがアクセスした際に「グループから外れました」を表示
