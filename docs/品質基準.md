## 1. セキュリティ

### 1.1 GitHub Security アラート（**ゼロトレランス**）

| アラート種別             | 重大度          | CI ゲート                               | SLA                                      |
| ------------------------ | --------------- | --------------------------------------- | ---------------------------------------- |
| Dependabot（依存脆弱性） | Critical / High | **マージブロック**                      | 検知から 24 時間以内に修正またはピン留め |
| Dependabot（依存脆弱性） | Medium          | 警告表示（ブロックしない）              | 検知から 7 日以内に対応                  |
| Dependabot（依存脆弱性） | Low             | 警告表示（ブロックしない）              | 次回スプリント内に対応                   |
| CodeQL SAST              | Critical / High | **マージブロック**                      | 検知から 24 時間以内に修正               |
| CodeQL SAST              | Medium / Low    | 警告表示（ブロックしない）              | 7 日以内に対応                           |
| Secret scanning          | 全重大度        | **マージブロック + 即時ローテーション** | 検知から 1 時間以内にシークレット無効化  |

**運用ルール:**

- Critical / High の未解決アラートが 1 件でも存在する場合、`main` ブランチへの PR マージを CI が自動的にブロックする。
- 誤検知（false positive）と判定した場合は、リポジトリ管理者（Kei）が GitHub UI から "Dismiss" し、理由をコメントに記載する。Dismiss なしのアラートを残したままマージすることは禁止。
- `npm audit --audit-level=high` を CI に組み込み、high 以上の脆弱性をビルドエラー扱いにする。

### 1.2 依存関係管理

| 項目                   | 基準                                                                                      |
| ---------------------- | ----------------------------------------------------------------------------------------- |
| パッケージ更新サイクル | Dependabot の自動 PR を週次でレビュー・マージ                                             |
| ピン留めポリシー       | `package.json` は `^` prefix を使用。CI では `package-lock.json` / `yarn.lock` 固定ビルド |
| 未使用依存             | `depcheck` で月次チェック。未使用パッケージは 30 日以内に削除                             |

### 1.3 認証・通信

| 項目                 | 基準                                                                             |
| -------------------- | -------------------------------------------------------------------------------- |
| JWT                  | アルゴリズム: RS256（非対称鍵）。`exp` 必須。アクセストークン有効期限 ≤ 15 分    |
| リフレッシュトークン | HTTPOnly Cookie のみ。`Secure` + `SameSite=Strict`                               |
| 通信                 | HTTPS 必須（TLS 1.2 以上）。HTTP → HTTPS リダイレクト                            |
| CORS                 | 許可オリジンは明示的に列挙。ワイルドカード `*` 禁止                              |
| レートリミット       | URL 取り込みエンドポイント: 10 req/min/user。同期 OCR 用エンドポイントは作らない |

## 2. コード品質

### 2.1 静的解析

| 項目            | 基準                                                               | CI ゲート                           |
| --------------- | ------------------------------------------------------------------ | ----------------------------------- |
| TypeScript      | `strict: true`（`noImplicitAny`, `strictNullChecks` 等を全て有効） | エラー 0 件でなければマージブロック |
| ESLint          | `@typescript-eslint/recommended` + `eslint-plugin-react-hooks`     | エラー 0 件でなければマージブロック |
| ESLint warnings | 警告は蓄積しない。新規 warning は 1 週間以内に修正またはルール調整 | 警告表示のみ（ブロックしない）      |
| Prettier        | コード整形の統一（CI で `prettier --check` を実行）                | 差分あればマージブロック            |

### 2.2 テストカバレッジ

| レイヤー                       | カバレッジ閾値 | 測定対象                                                 |
| ------------------------------ | -------------- | -------------------------------------------------------- |
| サービス層（ビジネスロジック） | ≥ 80% (lines)  | `apps/mobile/src/services/`, `apps/server/src/services/` |
| ユーティリティ関数             | ≥ 90% (lines)  | `packages/shared/src/utils/`, URL パーサー, OCR パーサー |
| UI コンポーネント              | ≥ 60% (lines)  | `apps/mobile/src/components/`                            |
| API エンドポイント             | ≥ 75% (lines)  | `apps/server/src/routes/`                                |
| 全体（プロジェクト合算）       | ≥ 70% (lines)  | —                                                        |

カバレッジが閾値を下回る PR はマージブロック。

### 2.3 コードサイズ・複雑性

| 指標         | 基準                                                           |
| ------------ | -------------------------------------------------------------- |
| 循環的複雑度 | 関数単位で ≤ 15。超過時は ESLint `complexity` ルールで警告     |
| 関数行数     | 1 関数 ≤ 80 行を目安（自動チェックなし、コードレビューで確認） |
| ファイル行数 | 1 ファイル ≤ 400 行を目安（自動チェックなし）                  |

### 2.4 OCR テスト設計と追跡性

OCR はネイティブ依存・画像品質・parser 連携が絡むため、実装単位とテスト単位を Trace ID で結び付ける。Trace ID はテスト名、E2E 結果、PR 説明に残す。

| Trace ID   | 対象                                                      | テスト種別    | 実装ファイル                                             | テストファイル                                            |
| ---------- | --------------------------------------------------------- | ------------- | -------------------------------------------------------- | --------------------------------------------------------- |
| OCR-SVC-01 | 画像前処理の入力/出力契約、品質警告                       | Unit          | `src/services/image-preprocess.service.ts`               | `src/services/__tests__/image-preprocess.service.test.ts` |
| OCR-SVC-02 | OCR raw text の正規化                                     | Unit          | `src/services/ocr.service.ts`                            | `src/services/__tests__/ocr.service.test.ts`              |
| OCR-SVC-03 | Source(type=ocr) 保存と RecipeRevision.sourceId 接続      | Service       | `src/services/source.service.ts`, `recipe.service.ts`    | `src/services/__tests__/recipe.service.test.ts`           |
| OCR-AGT-01 | A2: OCR provider 結果 → RecipeDraft                       | Unit          | `src/agents/ocr.agent.ts`                                | `src/agents/__tests__/ocr.agent.test.ts`                  |
| OCR-AGT-02 | A2: OCR_FAILED / PARSE_FAILED のエラー返却                | Unit          | `src/agents/ocr.agent.ts`                                | `src/agents/__tests__/ocr.agent.test.ts`                  |
| OCR-UI-01  | 権限確認、撮影/ギャラリー選択、画像プレビュー             | Component     | `app/(tabs)/recipes/import-ocr.tsx`                      | `app/(tabs)/recipes/__tests__/import-ocr.test.tsx`        |
| OCR-UI-02  | 失敗時の再撮影・手動入力 fallback                         | Component     | `app/(tabs)/recipes/import-ocr.tsx`                      | `app/(tabs)/recipes/__tests__/import-ocr.test.tsx`        |
| OCR-E2E-01 | 実機: OCR 入口 → プレビュー表示                           | E2E           | `e2e/android-e2e.mjs`                                    | `e2e/android-e2e-result.json`                             |
| OCR-E2E-02 | 実機: 低品質画像時の再撮影導線                            | E2E           | `e2e/android-e2e.mjs`                                    | `e2e/android-e2e-result.json`                             |
| OCR-SEC-01 | 同期 OCR でサーバー OCR / サーバー AI fallback を使わない | Unit / Review | `src/agents/ocr.agent.ts`, `src/services/ocr.service.ts` | `src/agents/__tests__/ocr.agent.test.ts`                  |

既存テストシナリオとの関連:

| 既存テスト                                               | OCR で再利用する理由                             | 対応 Trace ID          |
| -------------------------------------------------------- | ------------------------------------------------ | ---------------------- |
| `recipeTextParser.test.ts`: 見出し付き日本語レシピ       | OCR 正規化後の理想入力も同じ形式へ寄せる         | OCR-SVC-02, OCR-AGT-01 |
| `recipeTextParser.test.ts`: 見出しなし自由文             | OCR の改行崩れ・見出し欠落時の fallback と同じ   | OCR-SVC-02, OCR-AGT-01 |
| `recipeTextParser.test.ts`: sparse text placeholder      | OCR の低信頼結果をレビュー可能にする             | OCR-AGT-02             |
| `recipeTextNormalizer.test.ts`: local heuristic provider | OCR raw text の補正 provider と同じ境界          | OCR-SVC-02, OCR-AGT-01 |
| `recipe.service.test.ts`: createRecipe / getRecipeDetail | OCR 保存後も通常レシピとして取得できることを保証 | OCR-SVC-03             |
| `e2e/android-e2e.mjs`: T12 OCR スタブ画面                | OCR 実装後は T12 を OCR-E2E-01 / 02 に置換する   | OCR-E2E-01, OCR-E2E-02 |

OCR PR のマージ条件:

- `pnpm --filter mobile typecheck`
- `pnpm --filter mobile exec jest src/services/__tests__/ocr.service.test.ts src/agents/__tests__/ocr.agent.test.ts --runInBand`
- Source 保存を触る PR は `recipe.service.test.ts` も実行
- A2 契約を触る PR は `OCR-SEC-01` として、provider 未設定時にサーバー fallback しないことをテストで固定する
- UI まで触る PR は Android 実機またはエミュレーターで `e2e/android-e2e.mjs` を実行し、OCR Trace ID の結果を記録

## 3. パフォーマンス

### 3.1 モバイルアプリ（実機測定。低スペック端末: iPhone SE 第3世代 / Pixel 6a を基準）

| 指標                                                  | 目標値                   | 計測方法                           |
| ----------------------------------------------------- | ------------------------ | ---------------------------------- |
| コールドスタートアップ（アプリ起動 → ホーム画面表示） | < 3.0 秒                 | Flashlight / Maestro               |
| レシピ一覧表示（300 件）                              | < 500 ms                 | React Native Performance Monitor   |
| FTS 全文検索（食材名 / キーワード）                   | < 200 ms                 | SQLite `EXPLAIN QUERY PLAN` + 実測 |
| レシピ詳細画面遷移                                    | < 300 ms                 | React Native Performance Monitor   |
| 料理中モード ステップ切替                             | < 100 ms（体感で即時）   | 目視確認                           |
| JS スレッド フレームレート                            | ≥ 55 fps（スクロール中） | React Native DevTools              |
| OCR 処理（撮影 → テキスト抽出完了）                   | < 5 秒（端末内処理）     | 実測ログ                           |

### 3.2 サーバーサイド（p95 レイテンシ、単一ユーザー負荷）

| エンドポイント                            | p95 目標値 |
| ----------------------------------------- | ---------- |
| URL 取り込み（外部 fetch + JSON-LD 解析） | < 10 秒    |
| AI 構造化（OCR テキスト → レシピ JSON）   | < 15 秒    |
| レシピ CRUD API                           | < 200 ms   |
| 調理記録 POST                             | < 200 ms   |
| ヘルスチェック                            | < 50 ms    |

### 3.3 アプリバイナリサイズ

| プラットフォーム                | 最大サイズ（ダウンロードサイズ） |
| ------------------------------- | -------------------------------- |
| iOS（.ipa App Store 配信）      | ≤ 50 MB                          |
| Android（.aab Play Store 配信） | ≤ 30 MB                          |

超過した場合は `expo-bundle-analyzer` でモジュールを特定し、遅延読み込みまたは依存削除で対応する。

## 6. CI/CD ゲート一覧

### マージブロック条件（1 件でも該当すれば PR をマージ不可）

| #   | 条件                                                      |
| --- | --------------------------------------------------------- |
| S-1 | GitHub Security: Critical / High アラートが未解決         |
| S-2 | Secret scanning: シークレットが検知されている             |
| S-3 | `npm audit --audit-level=high` が high 以上の脆弱性を報告 |
| Q-1 | TypeScript コンパイルエラーが 1 件以上                    |
| Q-2 | ESLint エラーが 1 件以上                                  |
| Q-3 | Prettier フォーマット差分が存在する                       |
| Q-4 | テストカバレッジがレイヤー閾値を下回る                    |
| Q-5 | ユニットテスト / 結合テストが 1 件以上失敗                |

### 警告表示のみ（ブロックしない）

| #   | 条件                                                           |
| --- | -------------------------------------------------------------- |
| W-1 | GitHub Security: Medium / Low アラートが存在する               |
| W-2 | ESLint warnings が存在する                                     |
| W-3 | アプリバイナリサイズが目標値を超過（リリースブランチのみ計測） |
