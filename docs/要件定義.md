# だいどこ — 要件定義書

> 改訂: 2026-05-04  
> ステータス: Draft

---

## 1. プロダクト概要

**アプリ名:** だいどこ（臺所）  
**サブタイトル:** 家族で育てる、台所のレシピ手帳  
**対象プラットフォーム:** モバイル（iOS / Android）、将来的に Web

---

## 2. 問題定義

### 解決したい課題

クラシル・クックパッドなど既存のレシピアプリは「レシピを探す」用途に最適化されており、「自分・家族が実際に作ったもの」を記録・さかのぼる体験が乏しい。

- 同じレシピを何度作ったか分からない
- 誰がいつ作ったか・どんなアレンジをしたか残せない
- 家族間でレシピを共有・引き継ぐ仕組みがない
- 料理本・祖母の手書きメモなどアナログ資産をデジタルに持ち込めない

### 影響を受けるユーザー

- 料理をする家族の一員（主に 20〜50代）
- 「作りたいリスト」や「家族の得意料理」を整理したいと感じている人
- 祖母・親の味を記録・継承したいと考えている人

### 放置した場合のコスト

料理の記憶・家族の食の文化が散逸し、世代をまたいだ継承が難しくなる。

## 3. ゴール

| # | ゴール | 指標 |
|---|--------|------|
| G1 | ユーザーが自分・家族のレシピを一元管理できる | 登録レシピ数 / アクティブユーザーあたり |
| G2 | 調理記録が継続して蓄積される | 週次調理ログ入力率 ≥ 60% |
| G3 | 家族全員が同じレシピ帳を参照・更新できる | 家族招待成功率 ≥ 70% |
| G4 | レシピ追加の摩擦を最小化する | URL / OCR 取り込み成功率 ≥ 85% |
| G5 | 長期利用につながる愛着ある体験を提供する | 90日継続率 ≥ 40% |

---

## 4. スコープ外（Non-Goals）

| 項目 | 理由 |
|------|------|
| ソーシャル機能（他人のレシピを発見・フォロー） | コンセプトが「私設」であるため意図的に除外 |
| EC・食材購入連携 | v1 スコープを絞るため。v2 以降で検討 |
| カロリー・栄養素計算 | 別ドメインの複雑性を持ち込まないため |
| AIによる献立自動生成 | 過度な自動化はアナログ感を損なう |
| Web クラウド同期 | v1 はローカル DB のみ。v2 で追加 |

## 5. ユーザーストーリー

### 5.1 レシピ管理

- オーナーとして、URL を貼り付けるだけで外部レシピを取り込みたい。そうすれば手入力の手間を省けるから。
- オーナーとして、料理本や手書きメモを撮影してレシピ化したい。そうすれば家族のアナログ資産をデジタルに残せるから。
- オーナーとして、タイトル・材料・手順・覚書を自分で入力してレシピを作りたい。そうすれば完全オリジナルのレシピも保存できるから。
- オーナーとして、既存レシピに「バージョン」を派生させたい。そうすれば元のレシピを壊さずにアレンジを記録できるから。
- オーナーとして、レシピをカテゴリ・タグで分類したい。そうすれば蔵書が増えても目的のレシピを素早く見つけられるから。

### 5.2 調理記録

- 家族メンバーとして、「作った！」を 1 タップで記録したい。そうすれば調理の事実を簡単に残せるから。
- 家族メンバーとして、調理記録に写真・コメントを添付したい。そうすれば「この日はどんな仕上がりだったか」を振り返られるから。
- 家族メンバーとして、家族全員の調理タイムラインを見たい。そうすれば誰が何を作ったか一覧できるから。

### 5.3 調理サポート

- 料理中モードとして、手順を全画面・大文字で表示したい。そうすれば料理中に手を触れずに読み進められるから。
- 買い物モードとして、材料リストだけを表示しチェックしたい。そうすれば買い物時に材料の過不足を確認できるから。

### 5.4 家族共有

- オーナーとして、家族メンバーを招待したい。そうすれば同じレシピ帳を共有できるから。
- 家族メンバーとして、オーナーのレシピを閲覧・調理記録できるが編集・削除はできない。そうすれば誤ってレシピが壊れないから。
- 家族メンバーとして、既存レシピを元に自分用の派生バージョンを作りたい。そうすれば好みのアレンジを保存できるから。

## 6. 機能要件

### 6.1 Must-Have（P0）

#### レシピ管理
- **R01** レシピの CRUD（作成・閲覧・更新・削除）
  - 受け入れ基準: タイトル・材料・手順・覚書・写真・カテゴリ・タグを保存できる
- **R02** URL 取り込み
  - 受け入れ基準: クラシル・クックパッド等の URL を入力するとタイトル・材料・手順が自動抽出される（JSON-LD Recipe schema が存在する場合）
- **R03** 手動入力フォーム
  - 受け入れ基準: 全フィールドをフォームから入力・保存できる
- **R04** 調理記録（「作った！」）
  - 受け入れ基準: 日時・調理者が自動設定され、1 タップで記録できる
- **R05** レシピ一覧・検索
  - 受け入れ基準: キーワード・カテゴリ・タグで絞り込みができる

#### 閲覧・操作
- **R06** 料理中モード
  - 受け入れ基準: 手順を全画面大表示、スワイプで前後移動、画面常時 ON
- **R07** 買い物モード
  - 受け入れ基準: 材料リストをチェックボックス付きで表示できる

#### データ
- **R08** ローカル DB（SQLite / IndexedDB）
  - 受け入れ基準: アプリを再起動してもデータが保持される

### 6.2 Nice-to-Have（P1）

- **R09** 写真取り込み（OCR）: 端末カメラで撮影→テキスト自動抽出
- **R10** レシピのバージョン管理: 派生レシピを元レシピにリンク
- **R11** 家族タイムライン: 調理ログを時系列で一覧
- **R12** カレンダービュー: 月別に調理日を可視化
- **R13** 写真ギャラリー: 調理写真を一覧表示
- **R14** 家族招待・権限管理: メンバー招待フロー、オーナー判定ロジック
- **R15** 「作りたい」リスト: 個人・共有の両方をサポート

### 6.3 Future（P2）

- **R16** クラウド同期（v2）
- **R17** データエクスポート（PDF / JSON）
- **R18** Web 版

## 7. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | レシピ一覧は 300 件まで 200ms 以内に表示 |
| アクセシビリティ | 本文テキストコントラスト比 WCAG AA 以上（#DCC9A8 以上） |
| オフライン | 全機能がオフラインで動作する（v1 はローカルのみ） |
| データ保護 | 他デバイスへのデータ転送は v2 以降で対応。v1 はデバイス内のみ |
